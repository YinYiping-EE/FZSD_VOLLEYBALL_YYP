# 排球机器人串口数据包协议（补充完整版）
## robotArray - RX需接收数据
| Byte | 类型 | Data | 说明 |
| ---- | ---- | ---- | ---- |
| 0 | uint8 | 0xAA | 包头 |
| 1 | uint8 | mode | 执行模式 |
| 2 | uint8 | state | 接球模式下的阶段反馈 |
| 3-6 | float32 | X | 机器人位置，单位m |
| 7-10 | float32 | y | 机器人位置，单位m |
| 11-14 | float32 | yaw | 机器人航向角，单位度 |
| 15 | uint8 | Xor | 异或校验位 |
| 16 | uint8 | 0x55 | 包尾 |

### 枚举说明
#### mode说明
- 0：IDLE 待机等待指示
- 1：REMOTE 遥控接管中
- 2：SELF 自主接球中

#### state说明
- 0：waiting_plan 等待机械臂轨迹和机器人目标位置
- 1：received_plan 接收到规划
- 2：catching 自主接球中
- 3：over 结束接球

## planArray - TX发送内容
由于实际有多个命令：移动规划、机械臂规划，这里先列出总的框架：
| Byte | 类型 | Data | 说明 |
| ---- | ---- | ---- | ---- |
| 0 | uint8 | 0xAA | 包头 |
| 1 | uint8 | cmd | 指令位 |
| 2-3 | uint16 | len | 数据内容长度（在该字节之后数据字节长度，不包括校验位与包尾） |
| … |  |  |  |
| -2 | uint8 | Xor | 异或校验位 |
| -1 | uint8 | 0x55 | 包尾 |

### 枚举说明
#### cmd
- 0：move_plan 移动规划

### 移动规划（完整协议）
| Byte | 类型 | Data | 说明 |
| ---- | ---- | ---- | ---- |
| 0 | uint8 | 0xAA | 包头 |
| 1 | uint8 | cmd | 指令位 |
| 2-3 | uint16 | len | 数据内容长度，固定为4×5=20（注：含X、y、yaw、time四个float32数据） |
| 4-7 | float32 | X | 接球位置，单位m |
| 8-11 | float32 | y | 接球位置，单位m |
| 12-15 | float32 | yaw | 机器人朝向，单位为度 |
| 16-19 | float32 | time | 距离球到合适击球点的预测时间，单位s |
| 20 | uint8 | xor | 异或校验位 |
| 21 | uint8 | 0x55 | 包尾 |
